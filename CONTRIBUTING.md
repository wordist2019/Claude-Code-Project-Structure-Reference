# Contributing

Thank you for your interest in contributing to the Claude Code Project Structure Reference! 🎉

## How to Contribute

### Reporting Issues
- Open a GitHub Issue describing the problem
- Include steps to reproduce, expected behavior, and actual behavior

### Suggesting Improvements
- Open a GitHub Issue with the label `enhancement`
- Describe the improvement and why it would benefit the community

### Submitting a Pull Request

1. **Fork** the repository
2. **Create** a feature branch:
   ```bash
   git checkout -b feature/my-improvement
   ```
3. **Make** your changes following the conventions in `CLAUDE.md`
4. **Commit** using conventional commit format:
   ```bash
   git commit -m "feat(skills): add new data-analysis skill"
   ```
5. **Push** to your fork:
   ```bash
   git push origin feature/my-improvement
   ```
6. **Open** a Pull Request with a clear description of your changes

## Contribution Ideas

- 🆕 New skill templates (data analysis, documentation generator, etc.)
- 🔌 New MCP server configurations
- 🤖 New agent `.yml` examples
- 📖 Improved documentation
- 🐛 Bug fixes or corrections
- 🌍 Translations of the reference guide

## Code of Conduct

Be kind, constructive, and collaborative. This is a community resource — let's keep it welcoming for everyone.
